# 🗳️ Decentralized Voting System (DApp)

A full-stack, end-to-end decentralized application leveraging Blockchain for exact, immutable vote counting, intertwined with cutting-edge AI verification to ensure identity liveness and unique participation. It eliminates voter fraud while retaining voter privacy.

---

## 🏗️ Architecture & Tech Stack

This project is structured as a **Monorepo** spanning three integrated domains:

1. **Frontend (UI & Verification Module) — `/frontend`**
   - **Framework:** React.js powered by Vite
   - **Styling:** Tailwind CSS v4, Lucide React Icons
   - **Animation & 3D:** Framer Motion, `@react-three/fiber` (glassmorphism UI)
   - **AI/CV Integration:** 
     - `face-api.js` for facial geometric extraction and Euclidean liveness distance calculation.
     - `Tesseract.js` for on-device Client-Side OCR image data extraction (Aadhar/Passport matching).
   - **Web3 Interface:** `ethers.js` v6 for smart contract bridging.

2. **Backend (Metadata & Verification Node) — `/backend`**
   - **Server:** Node.js + Express
   - **Storage:** Local flat JSON DB (`users.json`) replacing heavy DB dependencies like MongoDB.
     - *Why?* To securely persist non-chain identity metadata linking without sacrificing DApp startup overhead. 

3. **Blockchain (Smart Contracts) — `/smart-contracts`**
   - **Framework:** Hardhat `v2.x` (Downgraded strictly to avoid Node 24 Windows ESM bugs)
   - **Language:** Solidity `^0.8.20`
   - **Network:** Designed for Local `hardhat node` but deployable to Sepolia / Mainnet.

## 🔥 Application Flow

### 1. Unified Registration (`/signup`)
- Voters register manually or use Google OAuth configurations. Records are securely managed by the Node.js backend.
- Prevents database collisions.

### 2. Administrator Panel (`/login?type=admin`)
- Using **MetaMask** wallet signature, an Admin can:
  - Create and timestamp local Elections.
  - Onboard verified Candidates directly to the Immutable Ledger via Smart Contract functions.
  - Conclude the election and review immutable tallies.

### 3. Voter Authorization (`/login?type=user`)
- **Step 1:** Enter primary credentials (User ID/Password).
- **Step 2:** **OCR Extraction:** Upload a government ID. Tesseract algorithms extract Name & mobile numbers locally on the user's hardware.
- **Step 3:** **Liveness Detection:** Enables webcam access. Extracts topological map from government ID photo and compares against live webcam bounding box. 
  - *Must achieve < 0.6 Euclidean distance confidence to proceed.*

### 4. Decentralized Voting Booth (`/vote`)
- Authorised voter casts their ballot.
- One wallet = One Vote (regulated dynamically within `Voting.sol`).

## 🚀 Execution & Setup Guide

Ensure you are running **Node 20+**. All services are intended to be run in parallel. 

#### A. Run Blockchain & Contracts
```bash
cd smart-contracts
npm install
# Start local chain:
npx hardhat node
# Deploy contract (In a separate terminal):
npx hardhat run scripts/deploy.js --network localhost
```
*Note: Copy the returned contract address to `frontend/src/config.js`.*

#### B. Run Backend API
```bash
cd backend
npm install
node server.js
```
*Note: This will auto-seed `admin001` and `voter001` within the newly created `users.json` mock database.*

#### C. Run Frontend (Vite)
```bash
cd frontend
npm install
npm run dev
```
*Note: The React App will now load at `http://localhost:5173`.*

## 🦊 Connecting MetaMask (Important)
Since standard accounts are dynamically loaded in Hardhat, follow this to view Admin configurations:
1. Copy one of the Private Keys generated by `npx hardhat node`.
2. Open MetaMask > Add Network > Custom RPC.
3. **RPC URL:** `http://127.0.0.1:8545`
4. **Chain ID:** `31337`
5. Import Account using the saved Private Key. This acts as the administrative owner.
